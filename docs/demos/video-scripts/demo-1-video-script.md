# Demo 1: The Power User Story - Video Script
## Finding the Perfect Tool with Advanced Search

**Video Duration:** 3-4 minutes
**Format:** Screencast with voiceover
**Recording Tool:** OBS, Loom, or Camtasia

---

## Pre-Recording Setup

### Technical Setup
- [ ] Screen resolution: 1920x1080 (Full HD)
- [ ] Browser: Chrome or Firefox (clean profile, no extensions showing)
- [ ] Application running at http://localhost:4200 or live URL
- [ ] Dataset: Code/Editors (default)
- [ ] Clear all filters and search
- [ ] Close unnecessary tabs and applications
- [ ] Turn off notifications (Do Not Disturb mode)
- [ ] Hide desktop icons/clean desktop
- [ ] Increase browser zoom if text too small (110-125%)

### Recording Settings
- [ ] Frame rate: 30 FPS minimum
- [ ] Audio: Clear microphone (test audio levels)
- [ ] Cursor highlighting enabled (helps viewers follow)
- [ ] Background music: None or very subtle
- [ ] Export format: MP4 (H.264 codec)

## Video Structure

### Introduction (0:00 - 0:20)
**On Screen:** Title card with "Demo 1: The Power User Story"
**Voiceover:**
> "In this demo, I'll show you how to use Awesome Comparisons to find the perfect AI code editor in just a few minutes. Let's say I'm looking for a code editor with three specific requirements: offline capability, support for my own API keys, and strong Git integration."

**Action:** Fade from title card to browser showing comparison page

---

### Scene 1: Initial View (0:20 - 0:35)
**On Screen:** Full comparison table with all code editors visible
**Voiceover:**
> "Here's our comparison of AI code editors. We're looking at over 30 different tools, which is a lot to review manually. Let's use the search and filter features to narrow this down."

**Action:**
- Hover over the comparison table briefly
- Scroll down to show the extent of the list
- Scroll back to top

**Camera/Cursor Notes:**
- Smooth scrolling
- Pause at top to show column headers
- Keep cursor visible

---

### Scene 2: First Filter - Offline Capability (0:35 - 1:05)
**On Screen:** Filter panel showing available criteria
**Voiceover:**
> "First, I need offline capability. I'll use the filter panel to narrow down to tools that support offline mode. Watch how the table automatically updates to show only matching tools."

**Action:**
1. Locate the filter panel (sidebar or dropdown)
2. Find "Local/Offline" or similar criterion
3. Click the dropdown or expand the filter
4. Select "Yes" or "Supported" option
5. Watch table filter

**Timing:**
- Clear demonstration of filter selection (3 seconds)
- Pause 2 seconds after filtering completes

**Voiceover (continued):**
> "Great! We've gone from 30+ tools down to just 8 that support offline mode. But we still need to narrow this down further."

**Action:**
- Briefly scroll through filtered results
- Show filter indicator/tag showing "Local/Offline: Yes"

**Camera/Cursor Notes:**
- Slow, deliberate cursor movement to filter control
- Clear demonstration of dropdown selection
- Show filter indicator after selection

**Voiceover:**
> "Next, I need 'Bring Your Own Key' support, often abbreviated as BYOK. This means I can use my own API keys instead of the provider's. Let me add this second filter."

**Action:**
1. In the filter panel, find "BYOK" or "Bring Your Own Key"
2. Click the dropdown or checkbox

4. Watch table update

**Alternative (if UI differs):**

2. Find "BYOK" option

**Timing:**

**Action:**
- Show both filter indicators/tags: "Local/Offline: Yes" and "BYOK: Yes"

**Camera/Cursor Notes:**
- Clear demonstration of adding second filter
- Show multiple filter indicators visible
- Emphasize narrowing effect

## Recording Tips

### Before Recording
1. **Script Rehearsal:** Read through script 2-3 times
2. **Dry Run:** Practice the clicks without recording
3. **Timing Check:** Ensure you can complete in 3-4 minutes
4. **Environment:** Quiet room, no interruptions
5. **Water:** Have water nearby for clear voice

### During Recording
1. **Pace:** Speak clearly and not too fast
2. **Pauses:** Leave 1-2 second pauses between major actions
3. **Cursor:** Move cursor deliberately, not frantically
4. **Breath:** Take natural breaths, can be edited out
5. **Mistakes:** If you make a mistake, pause 3 seconds, then restart that section

### After Recording
1. **Edit:** Remove long pauses, mistakes, "ums"
2. **Audio:** Normalize audio levels, remove background noise
3. **Captions:** Add captions/subtitles (improves accessibility)
4. **Transitions:** Add smooth transitions between scenes
5. **Music:** Add subtle background music if desired
